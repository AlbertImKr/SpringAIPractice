RAG (Retrieval Augmented Generation) 개념

RAG는 검색 증강 생성으로, 대규모 언어 모델의 출력을 최적화하는 프로세스입니다.
응답을 생성하기 전에 훈련 데이터 소스 외부의 신뢰할 수 있는 기술 자료를 참조합니다.

RAG의 주요 구성 요소:
1. 외부 데이터 생성: LLM의 원래 학습 데이터 외부에서 관련 문서를 벡터 저장소에 저장
2. 관련 문서 검색: 사용자 질문을 바탕으로 임베딩된 문서와의 유사도 기반 검색
3. LLM 프롬프트 증강: 검색된 문서를 사용자 질문과 결합하여 증강된 프롬프트 생성
4. 응답 생성: 증강된 프롬프트를 LLM에 전달하여 응답 생성
5. 외부 데이터 업데이트: 벡터 저장소의 문서를 정기적으로 업데이트

Spring AI의 RAG 구현:
- QuestionAnswerAdvisor: 기본 RAG 패턴 구현
- RetrievalAugmentationAdvisor: 모듈형 RAG 아키텍처 구현
- ETL 파이프라인: 문서 처리 및 벡터화

RAG의 장점:
- 최신 정보 제공
- 도메인 특화 지식 활용
- 정확성 향상
- 환각(hallucination) 감소
